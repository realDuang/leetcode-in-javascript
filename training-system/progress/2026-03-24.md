### Problem 40. Combination Sum II (3/4) ⚠️

| Blank | Answer | Correct | Result |
|-------|--------|---------|--------|
| ① Preprocess | `sort((a,b) => a-b)` | same | ✅ |
| ② Dedup | `remaining - candidates[i] < 0` | `i > start && candidates[i] === candidates[i-1]` | ❌ |
| ③ Push | `candidates[i]` | same | ✅ |
| ④ Recurse start | `i + 1` | same | ✅ |

**Key Mistake:** Wrote pruning instead of dedup. Dedup logic was completely missing.

**Lesson: `i > start` vs `i > 0`:**
- `i > start`: 同层不重复，跨层可重复 ✅
- `i > 0`: 任何位置和前一个相同就跳 → 误杀跨层合法重复 ❌
- 例: `[1,1,2] target=3` → `i > 0` 会错过合法组合 `[1,1,2]`

**Also:** 排序后超出 target 可以 `break` 而非 `continue`（后面更大，更不可能满足）

## Core Takeaways: Backtracking Three Patterns

| Type | End Condition | For Start | Recurse | Special |
|------|---------------|-----------|---------|---------|
| Permutation | `path.length === n` | `0` + used[] | path | used array |
| Subset | 无条件收集 | `start` | path, `i+1` | — |
| Combination Sum II | `remaining === 0` | `start` | path, `i+1`, rem | **sort + `i>start` dedup** |
