## Key Mistakes

### 221. Maximal Square
- **错误:** `dp[i][j] = dp[i-1][j-1] + 1`（仅当 matrix 相邻值为1时）
- **问题:** 只检查了相邻格的原始值是否为1，没考虑它们能构成多大正方形
- **反例:** 左下角为0时，不可能构成大正方形，但错误方程会给出过大的值
- **正确:** `dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1`
- **直觉:** 正方形受限于最短边，三个方向都必须至少 k-1 才能构成 k

## Core Takeaways

1. **Grid DP 基本模式:** `dp[i][j]` 依赖 上方 `dp[i-1][j]` 和左方 `dp[i][j-1]`
2. **221 特殊:** 正方形还要看左上 `dp[i-1][j-1]`，取 **min** 三方向 +1
3. **两序列 DP (LCS):** 相等 → `dp[i-1][j-1]+1`，不等 → `max(dp[i-1][j], dp[i][j-1])`
4. **状态定义区分:**
   - "前i个" → 答案 = `dp[m][n]`（LCS, Unique Paths）
   - "以i结尾" → 答案 = `max(dp[...])` (最大子数组和, LIS)
